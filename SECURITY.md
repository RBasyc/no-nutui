# 安全策略 (Security Policy)

## 支持的版本

我们仅为最新版本提供安全更新和支持。如果您使用的是旧版本，请尽快升级到最新版本以获取安全补丁。

| 版本系列 | 支持状态          | 说明                       |
| -------- | ----------------- | -------------------------- |
| 1.x.x    | :white_check_mark: | 当前维护版本，接收安全更新 |
| < 1.0    | :x:                | 已终止支持，请立即升级     |

> **注意**：我们会在发现安全漏洞后尽快发布修复版本。请持续关注项目的 Release 页面。

---

## 报告安全漏洞

如果您发现了安全漏洞，请**不要**公开创建 Issue 或在社交媒体上讨论。

### 报告流程

1. **发送安全报告邮件**
   - 邮件主题格式：`[安全漏洞] - 简要描述`
   - 请尽可能详细地描述漏洞信息：
     - 漏洞类型（XSS、SQL 注入、认证绕过等）
     - 受影响的版本
     - 漏洞复现步骤
     - 潜在影响范围
     - 建议的修复方案（如有）

2. **等待确认**
   - 我们会在 **48 小时内**确认收到您的报告
   - 初步评估后，我们会在 **7 个工作日内**告知您：
     - 漏洞是否被确认
     - 预计的修复时间表
     - 是否需要更多技术细节

3. **修复与发布**
   - 我们会优先处理安全漏洞
   - 修复完成后，我们会与您协调发布时间
   - 安全补丁发布前，我们会通知您

4. **致谢**
   - 在征得您同意后，我们会在安全公告中致谢您的贡献
   - 我们会评估漏洞严重程度，并酌情提供奖励或认可

### 漏洞严重程度分级

| 级别 | 描述 | 示例 |
| ---- | ---- | ---- |
| **严重** | 可远程执行代码、获取完全控制权 | RCE、SQL 注入、认证绕过 |
| **高危** | 可获取敏感数据或重要操作权限 | 数据泄露、越权操作 |
| **中危** | 有限的影响或需要特定条件 | XSS、CSRF、信息泄露 |
| **低危** | 影响范围小或利用难度高 | 轻微的信息泄露、UI 欺骗 |

## 安全最佳实践

### 开发者指南

#### 1. 认证与授权
- ✅ 始终使用 HTTPS 传输敏感数据
- ✅ Token 存储使用 `Taro.setStorageSync`，但避免存储在 URL 中
- ✅ 每个 API 请求都应携带有效的 Authorization Token
- ✅ 服务端必须验证 Token 有效性和权限范围

```typescript
// ✅ 正确示例
Taro.request({
  url: apiUrl,
  method: 'POST',
  header: {
    'Content-Type': 'application/json',
    Authorization: Taro.getStorageSync('token') || ''
  },
  data: requestData
})
```

#### 2. 数据验证
- ✅ 前端验证用于改善用户体验
- ✅ **服务端必须重新验证所有数据**
- ✅ 对用户输入进行转义和过滤

#### 3. 敏感信息处理
- ❌ 不要在代码中硬编码密钥、密码或 Token
- ✅ 使用环境变量管理敏感配置
- ✅ 日志中不应记录敏感信息（密码、Token 等）
- ✅ 错误信息不应泄露系统内部细节

#### 4. 第三方依赖
```bash
# 定期检查依赖漏洞
npm audit

# 自动修复可修复的漏洞
npm audit fix

# 更新依赖到最新安全版本
npm update
```

#### 5. 代码质量
- ✅ 使用 TypeScript 类型检查
- ✅ 启用 ESLint 和 Stylelint
- ✅ 使用 Husky 和 lint-staged 进行提交前检查

### 用户指南

#### 1. 账户安全
- 🔒 使用强密码（建议 8 位以上，包含字母、数字、符号）
- 🔒 定期更换密码
- 🔒 不要与他人共享账户
- 🔒 在公共设备上使用后及时退出登录

#### 2. 数据保护
- 💾 定期备份重要数据
- 💾 不要在不安全的网络环境下操作
- 💾 注意保护实验室敏感信息

#### 3. 应用更新
- 📱 及时更新到最新版本
- 📱 关注官方安全公告
- 📱 下载应用仅通过官方渠道

## 已知安全注意事项

### 当前版本限制

1. **客户端存储**
   - Token 存储在本地缓存中，设备丢失可能存在风险
   - 建议实施设备绑定或远程登出功能

2. **数据传输**
   - 确保 API 服务器配置了有效的 SSL/TLS 证书
   - 降级攻击防护依赖服务端配置

3. **小程序平台限制**
   - 微信小程序的安全审计由微信平台负责
   - 需要通过微信小程序安全资质审核
